* Layers
~nwchemc~ is deliberately small:

- ~include/nwchemc.h~ exposes the stable C ABI.
- ~schema/Potentials.capnp~ defines ~NWChemParams~, per-step ~ForceInput~,
  and the structured NWChem input stanzas.
- ~subprojects/c-capnproto~ provides the ~capnp-c~ runtime and compiler plugin.
- ~src/nwchemc_params.c~ opens serialized Cap'n Proto messages and returns the
  generated ~NWChemParams_ptr~ or ~ForceInput_ptr~ roots.
- ~src/nwchemc.c~ applies persistent session parameters to the embed bridge and
  calls the energy/gradient, force, or Hessian entry point.
- ~src/nwchem_embed_c_api.f90~ owns the ISO C binding layer and saved NWChem
  runtime configuration.
- ~src/nwchem_embed_legacy.F~ calls the fixed-form NWChem routines that set up
  RTDB, store numeric geometry and library basis records, and evaluate
  gradients and Hessians.

* Parameter Flow
The ABI receives a byte pointer and byte count.
~capn_init_mem()~ opens the flat message, ~capn_getp(capn_root(...))~ obtains
the generated ~NWChemParams_ptr~, and generated ~read_NWChemParams()~ calls are
used only as short-lived C views while the arena is alive.
Session force calls use the same pattern for generated ~ForceInput_ptr~ roots.

There is no persistent copy of the Cap'n Proto parameter tree in the C runtime.
The embed bridge stores the small Fortran strings and scalar values it needs to
evaluate repeated calls.
Coordinates, atomic numbers, charge, multiplicity, method, task state, and the
selected library basis are written through NWChem RTDB-facing APIs.
NWChem input parsing is reserved for caller-provided free-form stanzas whose
grammar is intentionally upstream NWChem input text.

* Long-Running Sessions
~nwchemc_session_create()~ applies an ~NWChemParams~ message once and keeps the
NWChem runtime state inside the session. ~nwchemc_session_calculate_forces()~
accepts a serialized ~ForceInput~ message for each step, so molecular dynamics,
optimizers, and socket-style drivers can update coordinates and the periodic
cell without rebuilding the full configuration carrier.

The C layer keeps reusable numeric scratch for per-step positions and atomic
numbers. It does not introduce a second configuration struct. ~ForceInput.pos~
and ~ForceInput.box~ are copied from the Cap'n Proto message into contiguous C
arrays with explicit unit conversion, then handed to the Fortran bridge.

The Fortran bridge stores geometry through NWChem geometry routines. When
~ForceInput.box~ is present, the row-major 3x3 cell vectors are converted to
Bohr and applied with ~geom_amatrix_set~, ~geom_isystype_set(3)~, and
~geom_P1_set~ before the task call. That keeps the structured path numeric for
geometry and cell data while leaving free-form text available for NWChem grammar
that lacks a stable direct entry point.

* MPI And Finalization
The embed layer records whether it owns the MPI/GA runtime.
~nwchemc_finalize()~ tears down that owned runtime through the legacy bridge.
When the process already runs under MPI, the MPI test path verifies that the
same ABI call works under ~mpirun -np 2~.

* Build Systems
Meson is the primary build.
CMake builds the same capnp-c generation path, stub/parser tests, optional
NWChem shared library, install targets, cmocka C tests, and CTest
serial/MPI gradient plus serial Hessian checks.
The release metadata keeps Meson, CMake, pixi, and changelog versions in
lockstep.

* NWChem Root Model
Embed mode links NWChem in-process and therefore validates the root path as a
build-tree layout. The required artifacts include ~src/config/nwchem_config.h~,
~src/tools/install/bin/ga-config~, ~src/stubs.o~, the source include
directories, Global Arrays install directories, and ~lib/$NWCHEM_TARGET~.

A package-manager runtime prefix with ~bin/nwchem~, ~lib~, and
~share/nwchem~ is a different shape. It can support an executable backend, but
it does not provide the metadata and object files that the direct C ABI uses for
the shared embed library.

* Contract
~NWChemParams~ is the user-facing parameter carrier.
Every caller sends the same Cap'n Proto schema to the C ABI.
The schema provides typed fields for common runtime controls and structured
stanzas for NWChem input blocks.

* Top-Level Fields
| Field | NWChem role |
|-------+-------------|
| ~basis~ | Basis name passed to the embed bridge, for example ~sto-3g~ or ~def2-svp~. |
| ~theory~ | Main task family, for example ~scf~ or ~dft~. Short aliases such as ~blyp~ select DFT. |
| ~scfType~ | SCF type or DFT exchange-correlation keyword used by the bridge. |
| ~charge~ | Molecular charge. |
| ~multiplicity~ | Spin multiplicity, with values below one normalized to singlet for the embed call. |
| ~task~ | User task label, normally ~gradient~ for the current ABI call. |
| ~title~ | Optional NWChem title/start prefix. |
| ~memoryMb~ | Memory request in MiB; zero keeps the NWChem/runtime default. |
| ~nwchemRoot~ | Optional runtime root hint exported as ~NWCHEM_TOP~. |
| ~scratchDir~ | Optional scratch directory exported as ~NWCHEM_SCRATCH_DIR~. |
| ~permanentDir~ | Optional permanent directory exported as ~NWCHEM_PERMANENT_DIR~. |
| ~inputStanzas~ | Structured NWChem input blocks. |
| ~inputBlocks~ | Raw NWChem input text applied after structured stanzas. |

* Configuration Model (typed vs escape hatch)
NWChem has a very large input/RTDB surface. This project models configuration in
layers inside a single Cap'n Proto schema (~schema/Potentials.capnp~ only—no
parallel TOML/JSON/YAML config language):

1. *Top-level ~NWChemParams~ fields* — embed/ABI-critical knobs consumed by
   ~nwchemc_set_params~ / embed bridge (~basis~, ~theory~, ~charge~, dirs, ...).
2. *Structured ~NWChemInputStanza~ kinds* — high-traffic input blocks with stable
   NWChem syntax and dedicated renderers in ~src/nwchemc_params.c~.
3. *Escape hatches* — everything else remains valid NWChem configuration via:
   - ~NWChemInputStanza.raw~ / ~inputBlocks~ (literal input fragments),
   - ~NWChemSetDirective~ (~set key value~ RTDB/input directives),
   - ~NWChemModuleStanza~ with ~NWChemModuleName~ + directives, or ~customName~,
   - ~NWChemGenericStanza~ for any ~name ... end~ block not yet specialized.

The intern inventory (~schema/inventory/nwchem_features.json~) mirrors every
module enum, stanza kind, top-level field, and public ABI symbol. Regenerate
with ~pixi run inventory-gen~; validate with ~pixi run inventory-check~.

* Structured Stanzas
| Stanza | Cap'n Proto type | NWChem text emitted |
|--------+------------------+---------------------|
| Generic block | ~NWChemGenericStanza~ | ~name~, one directive per line, then ~end~. |
| Directive | ~NWChemDirective~ | ~keyword arg0 arg1 ...~ inside a block. |
| RTDB set | ~NWChemSetDirective~ | ~set key [type] value...~. |
| DFT | ~NWChemDftStanza~ | ~dft~, typed DFT controls, extra directives, then ~end~. |
| SCF | ~NWChemScfStanza~ | ~scf~, vectors/maxiter/thresh/tol2e/noprint, directives, ~end~. |
| Task line | ~NWChemTaskStanza~ | ~task [theory] [operation] [ignore]~ (explicit deck task). |
| Driver | ~NWChemDriverStanza~ | ~driver~, maxiter/tight/loose/gmax/grms/xmax/xrms/xyz, directives, ~end~. |
| Property | ~NWChemPropertyStanza~ | ~property~, dipole/mulliken/quadrupole, directives, ~end~. |
| Basis block | ~NWChemBasisStanza~ | ~basis [spherical]~, optional ~* library~, ECP line, directives, ~end~. |
| Geometry meta | ~NWChemGeometryStanza~ | ~geometry~ header (units/symmetry/noautosym); atom coords via C ABI. |
| Known module | ~NWChemModuleStanza~ | A selected NWChem module block, tokenized directives, then ~end~. |
| Pseudopotential | ~NWChemPseudopotentialStanza~ | ~nwpw~, ~pseudopotentials~, library entries, extra directives, then ~end~. |
| Raw | ~NWChemInputStanza.raw~ | Literal NWChem text. |

* Direct embed apply
When ~apply_config_to_embed~ runs (embed builds only), typed
~NWChemDftStanza~, ~NWChemScfStanza~, ~NWChemDriverStanza~,
~NWChemPseudopotentialStanza~, and text-valued ~NWChemSetDirective~ fields are
applied outside rendered text where the bridge has a direct RTDB/API path.

1. ~nwchemc_params_extract_direct_dft~ reads ~xc~, ~direct~, smearing.
2. ~nwchemc_params_render_embed_input_blocks~ renders stanzas with promoted DFT
   fields omitted (extra ~directives~ inside ~dft~ still render as text).
3. ~nwchemc_embed_set_dft_direct~ stores ~xc~ as the effective DFT functional
   (~cfg_scf~ / ~dft:xc~), and the legacy RTDB setup writes ~dft:direct~,
   ~int2e:filesize~, ~int2e:memsize~, ~dft:smear_sigma~, and ~dft:spinset~.

~nwchemc_params_extract_direct_set_strings~ reads text ~set key value~ stanzas.
The full deck renderer still emits ~set key value~. The embed renderer omits
those text-valued set lines and ~nwchemc_embed_set_rtdb_strings~ stores them
through ~rtdb_cput~ before method state is applied.

~NWChemSetDirective.valueType~ and ~values~ carry structured RTDB set values:
~text~ maps to NWChem's ~string~ form and ~rtdb_cput~, ~double~ maps to
~MT_DBL~, ~integer~ maps to ~MT_INT~, and ~logical~ maps to ~MT_LOG~.
~nwchemc_params_extract_direct_set_values~ and
~nwchemc_embed_set_rtdb_values~ keep those typed/list values out of
~nw_inp_from_character~ for gradient, Hessian, and dipole calls. The older single
~value~ field remains valid for one text value. Literal NWChem text remains
available for parser-specific ~set~ syntax that does not fit this typed value
carrier.

Callers still send one ~NWChemParams~ message; there is no second public config
struct.

* DFT Controls
~NWChemDftStanza~ provides typed fields for common DFT setup:

| Field | NWChem text |
|-------+-------------|
| ~xc~ | ~xc <value>~ |
| ~direct~ | ~direct~ |
| ~smearing.sigmaHartree~ | ~smear <sigma> fixsz~ or ~smear <sigma> nofixsz~ |
| ~directives~ | Extra tokenized DFT directives. |

Fermi smearing and related occupancy controls can also be expressed with
~NWChemSetDirective~, for example RTDB keys under ~dft:*~, or with explicit DFT
directives when the NWChem input syntax requires them.

* Module Coverage
~NWChemModuleStanza~ gives callers a typed module selector instead of a free
text block name. It covers the curated NWChem input-handler set represented in
~NWChemModuleName~, including molecular, relativistic, DFT/TDDFT, NWPW/PSPW,
optimization, property, solvation, task, utility, and XTB input handlers.

Each module stanza carries ~NWChemDirective~ entries, so a language binding can
construct NWChem input without concatenating block text. ~customName~ remains
available for NWChem modules not yet enumerated by the schema.
~tests/test_schema_module_coverage.py~ guards the enum against losing the
handler names derived from the local NWChem source tree.

* SCF Controls
~NWChemScfStanza~ covers common HF/SCF setup that complements top-level
~scfType~ (rhf/uhf/...). Typed fields:

| Field | NWChem text |
|-------+-------------|
| ~vectorsInput~ | ~vectors input <path>~ |
| ~vectorsOutput~ | ~vectors output <path>~ |
| ~maxiter~ | Full input emits ~maxiter <n>~; embed writes ~scf:maxiter~ or ~dft:iterations~ directly. |
| ~thresh~ | Full input emits ~thresh <value>~; embed writes ~scf:thresh~ directly. |
| ~tol2e~ | Full input emits ~tol2e <value>~; embed writes ~scf:tol2e~ or ~dft:tol2e~ directly. |
| ~noprint~ | ~noprint~ |
| ~directives~ | Extra SCF directives (convergence, semidirect, ...). |

* Task / Driver / Property / Basis / Geometry Stanzas
- *Task* (~NWChemTaskStanza~): emit an explicit ~task theory operation~ line when
  building a multi-task deck. Embed callers usually rely on top-level ~theory~ +
  ~task~ only; this stanza is for full-input generation. Embed rendering omits
  task stanzas because the C ABI entry point writes ~task:theory~ and
  ~task:operation~ directly for the requested energy, gradient, force, or
  Hessian call. Dipole calls write the energy operation and read the dipole
  vector from RTDB.
- *Driver* (~NWChemDriverStanza~): geometry optimization controls. Full input
  emits ~maxiter <n>~, ~tight~, ~loose~, and explicit ~gmax~ / ~grms~ /
  ~xmax~ / ~xrms~ values. Embed writes ~driver:nptopt~ and the
  ~driver:gmax_tol~ / ~driver:grms_tol~ / ~driver:xrms_tol~ /
  ~driver:xmax_tol~ values directly, and keeps ~xyz~ path plus extra directives
  as driver text.
- *Property* (~NWChemPropertyStanza~): common property keywords (~dipole~,
  ~mulliken~, ~quadrupole~) plus extra directives.
- *Basis* (~NWChemBasisStanza~): structured ~basis~ block when top-level ~basis~
  string alone is insufficient (spherical, segment library line, ECP token,
  per-element directives). Embed still applies top-level ~basis~ via the bridge.
- *Geometry* (~NWChemGeometryStanza~): block *metadata only* (units, symmetry,
  noautosym/noautoz/center). Nuclear coordinates always come from the C ABI
  (~positions_ang~, ~atomic_numbers~), not Cap'n Proto atom lists.

* Pseudopotentials
~NWChemPseudopotentialStanza~ provides a structured representation for NWPW
pseudopotential selection:

| Field | NWChem text |
|-------+-------------|
| ~entries[].element~ | Element symbol such as ~Si~ or ~O~. |
| ~entries[].libraryType~ | ~library~, ~pspw_library~, ~paw_library~, ~cpi~, or ~teter~. |
| ~entries[].libraryName~ | Library or file token used by NWChem. |
| ~entries[].allElements~ | NWChem ~*~ default entry for every element. |
| ~directives~ | Extra tokenized NWPW directives emitted after the pseudopotential block. |

The renderer emits these entries inside an ~nwpw~ block with a nested
~pseudopotentials~ section. For example, an entry with ~element = "Si"~,
~libraryType = library~, and ~libraryName = "sg15"~ renders as
~Si library sg15~.
When ~allElements~ is true, the renderer and embed direct path use NWChem's
~*~ target, which maps to fallback RTDB keys such as ~*:pspw_library~,
~*:pspw_library_type~, and ~*:paw_library~.

* Coverage Model
Typed fields cover the ABI-critical method, charge, multiplicity, directory,
DFT smearing controls, and NWPW pseudopotential library selection.
Known module stanzas represent the normal NWChem block vocabulary with enum
selection and tokenized directives.
The generic stanza and directive model represents arbitrary named NWChem blocks
without requiring a raw string.
The pseudopotential stanza covers library/file PSP entries (intern
~stanza.pseudopotential~).
The raw block is kept for exact upstream syntax, generated inputs, and NWChem
keywords whose grammar is better preserved as literal input.
The intern inventory guarantees every schema enum/field/stanza/ABI symbol has a
documented intern row with stub vs embed applicability.

* Upstream / Reference Coverage Boundary
The curated ~NWChemModuleName~ enum tracks NWChem input-handler names (see
~tests/test_schema_module_coverage.py~). Method-specific option dictionaries
(every DFT grid keyword, every TCE flag, every NWPW PSPW/BAND/PAW knob) are
*not* fully typed field-by-field; callers express them with:

- ~NWChemDirective~ lists inside DFT/SCF/module/generic stanzas,
- ~NWChemSetDirective~ for RTDB-style ~set~ lines,
- ~raw~ / ~inputBlocks~ for verbatim NWChem input.

This is intentional: the Cap'n Proto contract stays logical and maintainable
while remaining complete for NWChem configuration via escape hatches.

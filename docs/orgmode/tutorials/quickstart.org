* Stub and Parser Build
The default build exercises the stable C ABI and the Cap'n Proto parser without
linking a local NWChem tree.

#+begin_src bash
meson setup build -Dwith_tests=true
meson compile -C build
meson test -C build --print-errorlogs
#+end_src

This configuration builds the stub library, compiles the vendored ~capnp-c~
runtime, generates C readers for ~schema/Potentials.capnp~, and runs the parser
and stub ABI tests with cmocka. The parser test uses a normal Cap'n Proto text
fixture encoded by the ~capnp~ CLI.

cmocka is resolved through pkg-config. The pixi environment installs cmocka
alongside Meson, CMake, Cap'n Proto, and the compilers.

* NWChem Embed Build
The real embed build needs a source/build tree with the NWChem archives, Global
Arrays tools, source headers, ~src/stubs.o~, and target-specific configuration
files.

Build ~src/stubs.o~ through NWChem if it is missing:

#+begin_src bash
export NWCHEM_TOP=/path/to/nwchem
export NWCHEM_TARGET=LINUX64
make -C "$NWCHEM_TOP/src" stubs.o
#+end_src

Use a static ~nwchemc~ build for NWChem trees that ship non-PIC static
archives:

#+begin_src bash
meson setup build-nwchem \
  -Dwith_nwchem=true \
  -Dnwchem_root="$NWCHEM_TOP" \
  -Dnwchem_target="$NWCHEM_TARGET" \
  -Dwith_tests=true \
  -Ddefault_library=static
meson compile -C build-nwchem
meson test -C build-nwchem --print-errorlogs
#+end_src

The embed test suite exercises the primary calculation surface: energy,
gradient, forces, Hessian, dipole, polarizability, quadrupole, optimize,
frequencies, plus serialized session results, RTDB geometry/basis state, and
pseudopotential state against the same ~NWChemParams~ and ~ForceInput~ Cap'n
Proto carriers. Dedicated probes include ~test_nwchem_energy_forces~,
~test_nwchem_postscf_energy~, ~test_nwchem_optimize_freq~, and
~test_nwchem_primary_props~ (each can write ~NWCHEMC_COMPARE_JSON~ for the CLI
compare driver). Stress tests require an NWChem tree linked with NWPW/PSPW.

Shared ~libnwchemc.so~ builds are available when the selected NWChem archives
are PIC-compatible. If the shared link probe reports an ELF relocation error,
use the static build above or rebuild the selected NWChem target with PIC flags.

The conda-forge ~nwchem~ package provides an executable/runtime prefix. That is
not the same shape as the embed build tree: the in-process ABI needs NWChem
build metadata such as ~src/config/nwchem_config.h~ and
~src/tools/install/bin/ga-config~. Conda-forge compiler packages may also pull a
Linux sysroot into a development environment; that sysroot is compiler ABI
plumbing and is not an NWChem root.

* CLI and Embed Comparison
The integration matrix compares selected ~tests/integration/nw/*.nw~ inputs
against the embedded C ABI output.

#+begin_src bash
python3 tools/compare_nwchem_cli.py \
  --out-dir build/nwchem-compare-embed \
  --embed-build build-nwchem \
  --require-cli \
  --require-embed
#+end_src

When both the NWChem executable and the embed test binary need an MPI launcher,
pass both launch commands:

#+begin_src bash
python3 tools/compare_nwchem_cli.py \
  --out-dir build/nwchem-compare-embed \
  --embed-build build-nwchem \
  --require-cli \
  --require-embed \
  --nwchem-command "mpirun -np 2 $NWCHEM_TOP/bin/$NWCHEM_TARGET/nwchem" \
  --embed-command "mpirun -np 2"
#+end_src

The report files are written to the selected output directory as
~nwchem_compare_report.txt~ and ~nwchem_compare_report.json~.

The contract for which quantities are compared is
~tests/integration/task_matrix.json~ (~embed_compare~, ~embed_meson_test~,
tolerances). Rows cover energy/forces across SCF and post-SCF theories,
optimize (energy + optimized positions), frequencies (energy + magnitudes),
dipole, quadrupole, and Hessian (via the CLI ~.hess~ file). Polarizability is
proven on the embed ABI with a non-trivial magnitude gate but is not inventively
parsed from energy-only CLI logs. Authoritative embed-vs-CLI runs use a real
NWChem embed build and CLI on the same host (project practice: ~rg.terra~);
laptop-only CLI is not a substitute when claiming embed parity.

* CMake Build
The CMake build exposes the same stub/parser path and the real NWChem embed
path.

#+begin_src bash
cmake -S . -B .build-tmp/cmake-stub -DNWCHEMC_BUILD_TESTS=ON
cmake --build .build-tmp/cmake-stub --parallel 4
ctest --test-dir .build-tmp/cmake-stub --output-on-failure
#+end_src

For a real NWChem build:

#+begin_src bash
cmake -S . -B .build-tmp/cmake-nwchem \
  -DNWCHEMC_BUILD_TESTS=ON \
  -DNWCHEMC_WITH_NWCHEM=ON \
  -DNWCHEMC_NWCHEM_ROOT="$NWCHEM_TOP" \
  -DNWCHEMC_NWCHEM_TARGET="$NWCHEM_TARGET" \
  -DNWCHEMC_BUILD_SHARED=OFF
cmake --build .build-tmp/cmake-nwchem --parallel 4
ctest --test-dir .build-tmp/cmake-nwchem --output-on-failure
#+end_src

* MPI Test
Configure the real-test rank count when the selected NWChem target requires an
MPI launcher for all in-process tests. The ~OMPI_MCA_*~ variables are Open MPI
transport controls and are only needed on hosts that require them:

#+begin_src bash
meson setup build-nwchem-mpi \
  -Dwith_nwchem=true \
  -Dnwchem_root="$NWCHEM_TOP" \
  -Dnwchem_target="$NWCHEM_TARGET" \
  -Dwith_tests=true \
  -Ddefault_library=static \
  -Dnwchem_test_mpi_ranks=2
env OMPI_MCA_pml=ob1 OMPI_MCA_btl=self,tcp \
  meson test -C build-nwchem-mpi --print-errorlogs --num-processes 2
#+end_src

CMake exposes the same launcher setting:

#+begin_src bash
cmake -S . -B .build-tmp/cmake-nwchem-mpi \
  -DNWCHEMC_BUILD_TESTS=ON \
  -DNWCHEMC_WITH_NWCHEM=ON \
  -DNWCHEMC_NWCHEM_ROOT="$NWCHEM_TOP" \
  -DNWCHEMC_NWCHEM_TARGET="$NWCHEM_TARGET" \
  -DNWCHEMC_BUILD_SHARED=OFF \
  -DNWCHEMC_NWCHEM_TEST_MPI_RANKS=2
cmake --build .build-tmp/cmake-nwchem-mpi --parallel 4
env OMPI_MCA_pml=ob1 OMPI_MCA_btl=self,tcp \
  ctest --test-dir .build-tmp/cmake-nwchem-mpi --output-on-failure
#+end_src

The embedded runtime initializes and finalizes the MPI/GA state only when it
owns that runtime.

* Stub And Parser Build
The default build exercises the stable C ABI and the Cap'n Proto parser without
linking a local NWChem tree.

#+begin_src bash
meson setup build -Dwith_tests=true
meson compile -C build
meson test -C build --print-errorlogs
#+end_src

This configuration builds the stub library, compiles the vendored ~capnp-c~
runtime, generates C readers for ~schema/Potentials.capnp~, and runs the parser
test against a normal Cap'n Proto text fixture encoded by the ~capnp~ CLI.

* NWChem Embed Build
The real embed build needs a source/build tree with the NWChem archives, Global
Arrays tools, source headers, ~src/stubs.o~, and target-specific configuration
files.

#+begin_src bash
export NWCHEM_TOP=/path/to/nwchem
meson setup build-nwchem \
  -Dwith_nwchem=true \
  -Dnwchem_root="$NWCHEM_TOP" \
  -Dnwchem_target=LINUX64 \
  -Dwith_tests=true
meson compile -C build-nwchem
meson test -C build-nwchem --print-errorlogs
#+end_src

The NWChem archives must be linkable into a shared object.
On ELF targets that usually means building the NWChem object code with
position-independent code flags.

The conda-forge ~nwchem~ package provides an executable/runtime prefix. That is
not the same shape as the embed build tree: the in-process ABI needs NWChem
build metadata such as ~src/config/nwchem_config.h~ and
~src/tools/install/bin/ga-config~. Conda-forge compiler packages may also pull a
Linux sysroot into a development environment; that sysroot is compiler ABI
plumbing and is not an NWChem root.

* CMake Build
The CMake build exposes the same stub/parser path and the real NWChem embed
path.

#+begin_src bash
cmake -S . -B .build-tmp/cmake-stub -DNWCHEMC_BUILD_TESTS=ON
cmake --build .build-tmp/cmake-stub --parallel 4
ctest --test-dir .build-tmp/cmake-stub --output-on-failure
#+end_src

For a real NWChem build:

#+begin_src bash
cmake -S . -B .build-tmp/cmake-nwchem \
  -DNWCHEMC_BUILD_TESTS=ON \
  -DNWCHEMC_WITH_NWCHEM=ON \
  -DNWCHEMC_NWCHEM_ROOT="$NWCHEM_TOP" \
  -DNWCHEMC_NWCHEM_TARGET=LINUX64
cmake --build .build-tmp/cmake-nwchem --parallel 4
ctest --test-dir .build-tmp/cmake-nwchem --output-on-failure
#+end_src

* MPI Test
When ~mpirun~ is available, the build registers an MPI test that runs the same
C ABI energy/gradient executable under two ranks:

#+begin_src bash
meson test -C build-nwchem --suite mpi --print-errorlogs
ctest --test-dir .build-tmp/cmake-nwchem -L mpi --output-on-failure
#+end_src

The embedded runtime initializes and finalizes the MPI/GA state only when it
owns that runtime.

* Stub And Parser Build
The default build exercises the stable C ABI and the Cap'n Proto parser without
linking a local NWChem tree.

#+begin_src bash
meson setup build -Dwith_tests=true
meson compile -C build
meson test -C build --print-errorlogs
#+end_src

This configuration builds the stub library, compiles the vendored ~capnp-c~
runtime, generates C readers for ~schema/Potentials.capnp~, and runs the parser
test against a normal Cap'n Proto text fixture encoded by the ~capnp~ CLI.

* NWChem Embed Build
The real embed build needs a source or install tree with the NWChem archives,
Global Arrays tools, and target-specific configuration files.

#+begin_src bash
export NWCHEM_TOP=/path/to/nwchem
meson setup build-nwchem \
  -Dwith_nwchem=true \
  -Dnwchem_root="$NWCHEM_TOP" \
  -Dnwchem_target=LINUX64 \
  -Dwith_tests=true
meson compile -C build-nwchem
meson test -C build-nwchem --print-errorlogs
#+end_src

The NWChem archives must be linkable into a shared object.
On ELF targets that usually means building the NWChem object code with
position-independent code flags.

* MPI Test
When ~mpirun~ is available, Meson registers an MPI test that runs the same C ABI
energy/gradient executable under two ranks:

#+begin_src bash
meson test -C build-nwchem --suite mpi --print-errorlogs
#+end_src

The embedded runtime initializes and finalizes the MPI/GA state only when it
owns that runtime.

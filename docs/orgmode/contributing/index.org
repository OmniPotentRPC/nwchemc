* Development Checks
Run the default parser and stub checks before changing the ABI or schema:

#+begin_src bash
meson setup build -Dwith_tests=true
meson test -C build --print-errorlogs
#+end_src

For a configured NWChem tree, run the real embed checks with the same build
shape used by the target NWChem archives. Many NWChem trees provide non-PIC
static archives, so the static Meson build is the first path to try:

#+begin_src bash
export NWCHEM_TOP=/path/to/nwchem
export NWCHEM_TARGET=LINUX64
meson setup build-nwchem \
  -Dwith_nwchem=true \
  -Dnwchem_root="$NWCHEM_TOP" \
  -Dnwchem_target="$NWCHEM_TARGET" \
  -Dwith_tests=true \
  -Ddefault_library=static
meson compile -C build-nwchem
meson test -C build-nwchem --print-errorlogs
#+end_src

If the selected NWChem target requires an MPI launcher for in-process tests,
configure the launch rank count. The ~OMPI_MCA_*~ variables are Open MPI
transport controls and are only needed on hosts that require them:

#+begin_src bash
meson setup build-nwchem-mpi \
  -Dwith_nwchem=true \
  -Dnwchem_root="$NWCHEM_TOP" \
  -Dnwchem_target="$NWCHEM_TARGET" \
  -Dwith_tests=true \
  -Ddefault_library=static \
  -Dnwchem_test_mpi_ranks=2
env OMPI_MCA_pml=ob1 OMPI_MCA_btl=self,tcp \
  meson test -C build-nwchem-mpi --print-errorlogs --num-processes 2
#+end_src

Use the CLI comparison driver when an NWChem executable is also available:

#+begin_src bash
python3 tools/compare_nwchem_cli.py \
  --out-dir build/nwchem-compare-embed \
  --embed-build build-nwchem \
  --require-cli \
  --require-embed
#+end_src

* Documentation
Documentation source lives under ~docs/orgmode~.
The docs build exports orgmode to RST, runs Doxygen over the C/Fortran surface,
converts XML through doxyrest, and builds Sphinx HTML.

#+begin_src bash
pixi run -e docs docbld
#+end_src

The parser uses ~capnp-c~ and keeps Cap'n Proto as the language-neutral
configuration carrier. The runtime does not copy the parameter tree into a
second public configuration struct. One-shot ~*_from_config~ entry points
unwrap ~PotentialConfig.nwchem~, create the same configured session state as
~nwchemc_session_create_from_config~, validate output capacity before applying
configuration, and then call the corresponding session calculation.
Long-running callers can keep one ~NWChemParams~ configuration in a session and
send per-step ~ForceInput~ Cap'n Proto messages for coordinates, atomic
numbers, length units, optional cell vectors, optional charge or multiplicity
overrides, and unit-aware
~PotentialResult~ output.
The first accepted session evaluation establishes the session topology from
atom count and ordered atomic numbers. Later session evaluations reject atom
count or species changes before entering the embed bridge, while still allowing
coordinate, unit, and cell-vector changes.

* RTDB Path
The embedded Fortran layer writes numeric geometry through NWChem geometry APIs
and writes Gaussian library basis data through NWChem basis APIs. Session
~ForceInput.box~ values are applied as numeric 3x3 cells through
~geom_amatrix_set~, ~geom_isystype_set(3)~, and ~geom_P1_set~. The bridge then
stores charge, multiplicity, method, and task state through RTDB-facing calls
before calling ~task_energy~, ~task_gradient~, or ~task_hessian~. The dipole
ABI calls use ~task_energy~ and then read ~task:dipole~ from RTDB, falling back
to method-specific dipole keys where NWChem stores them.
The polarizability ABI calls use the same numeric geometry and basis setup, run
the energy task, set AO response property defaults where the caller has not
provided them, run the property task, and read the 12-value ~aoresponse:alpha~
vector from RTDB. The vector order is frequency, Cartesian components, three
eigenvalues, isotropic average, and anisotropy.
The quadrupole ABI calls use the same numeric geometry and basis setup, run
~task_energy~, load NWChem geometry and AO basis handles from RTDB, initialize
the integral package, and call ~hnd_mtpole2~ directly. The returned buffer holds
the six total traceless quadrupole components in ~xx, xy, xz, yy, yz, zz~ order.
The stress ABI calls store the same geometry, basis, pseudopotential, and
method state, set ~task:operation~ to ~stress~, call the generic NWChem task
dispatcher, and read ~pspw:stress~, ~band:stress~, ~paw:stress~, or
~ofpw:stress~ from RTDB.

NWChem modules read and mutate RTDB entries inside task routines and module
input handlers. Direct RTDB writes must therefore preserve this order:

1. Initialize GA/MA/RTDB and file-directory state.
2. Store numeric geometry and basis records.
3. Apply caller-provided structured or raw NWChem stanzas.
4. Store scalar method/task keys.
5. Call the NWChem task routine.
6. Read task output from RTDB or the NWChem task output file.

~nw_inp_from_character~ remains reserved for caller-provided NWChem stanzas.
Geometry coordinates and library basis selection are not routed through text
input. Typed SCF scalar controls (~maxiter~, ~thresh~, and ~tol2e~), driver
~maxiter~, driver convergence presets (~tight~ / ~loose~), and explicit driver
~gmax~ / ~grms~ / ~xmax~ / ~xrms~ tolerances are promoted into direct RTDB
writes for embed calculations. Structured NWPW cutoff controls
(~energyCutoff~, ~wavefunctionCutoff~, ~ewaldRcut~, and ~ewaldNcut~) are also
promoted to the NWPW RTDB keys used by ~nwpw_input.F~. Promoted fields are
omitted from embed-rendered text when no non-promoted syntax remains.
Structured NWPW state controls and simple named ~exchangeCorrelation~ values
use the same direct RTDB path. Born-Oppenheimer NWPW controls with stable
RTDB keys are promoted in the same pass. NWPW execution controls with stable
RTDB keys are also promoted directly. Output and motion filename fields are
promoted to the paired CPMD/NWPW RTDB string keys. Fractional orbital and smear
controls write the matching ~nwpw:fractional_*~ keys directly.
Virtual orbital counts, the ~virtual~ alias, LCAO skip/use mode, and Ewald
grid dimensions are promoted to direct ~nwpw:*~ keys. Nose-Hoover thermostat
controls and the ~temperature~ alias write the paired ~cpmd:*~ / ~nwpw:*~
nose, restart, period, temperature, and chain-length keys directly. NWPW atom
electric-field controls write the matching
~nwpw:atom_efield~ and ~nwpw:atom_efield_grad~ logical keys. NWPW Mulliken
analysis controls write the five ~*:mulliken~ logical keys and the
~nwpw:mulliken_kawai~ logical key. Periodic dipole mode writes
~nwpw:periodic_dipole~ directly. External electric-field controls write
~nwpw:efield~, ~nwpw:efield_vector~, optional ~nwpw:efield_center~, and
~nwpw:efield_type~ directly. Smooth cutoff controls write the
~nwpw:smooth_cutoff~ double pair directly. Cutoff boot wavefunction controls
write the ~nwpw:cutoff_boot_psi~ logical key directly. Fast-erf controls write
the ~nwpw:fast_erf~ logical key directly. Dipole motion controls write the
~nwpw:dipole_motion~ logical key and optional
~nwpw:dipole_motion_filename~ text key directly. Rho symmetry controls write
the ~nwpw:rho_use_symmetry~ logical key directly. FMM controls write
~nwpw:fmm~, ~nwpw:fmm_lmax~, and ~nwpw:fmm_lr~ directly. Born solvation
controls write ~nwpw:born~, ~nwpw:born_dielec~, ~nwpw:born_relax~, and
~nwpw:born_vradii~ directly; the Cap'n Proto radii field uses Angstrom
input values and the direct RTDB path stores Bohr values. CPMD property
and grid-comparison controls write ~nwpw:cpmd_properties~ and
~nwpw:use_grid_cmp~ directly. Director controls write
~nwpw:use_director~ and optional ~nwpw:director_filename~ directly. Cell
expansion, numeric mapping, and named mapping-alias controls write
~nwpw:cell_expand~ and ~nwpw:mapping~ directly. Rotation and multipole
controls write ~nwpw:rotation~ and ~nwpw:lmax_multipole~ directly. Structured
simulation-cell controls write boundary conditions, 3x3 cell vectors, FFT
grids, small grids, box delta, box orientation, and box-type state to the
matching ~<cell>:*~ RTDB keys.

* Pickup Tasks
** Long-Running Step Calls
~nwchemc_session_calculate_forces~,
~nwchemc_session_calculate_result~,
~nwchemc_session_calculate_hessian~,
~nwchemc_session_calculate_dipole~,
~nwchemc_session_calculate_polarizability~,
~nwchemc_session_calculate_quadrupole~, and
~nwchemc_session_calculate_stress~ are C ABI entry points for socket-style,
optimizer-style, property, stress, and vibrational callers. They take the
session's persistent ~NWChemParams~ and a per-call serialized ~ForceInput~
message, validate atom counts and output buffer sizes, convert coordinates and
cells to the C ABI units, call the embed bridge, and return either native
NWChem buffers or a serialized ~PotentialResult~ in the caller's requested
units.
~nwchemc_potential_result_size_for_force_input~ gives callers the
~PotentialResult~ byte count from the per-step ~ForceInput~ without evaluating
NWChem.
~nwchemc_calculate_result~ is the one-shot convenience wrapper around the same
~NWChemParams + ForceInput -> PotentialResult~ path. Multi-step callers should
reuse a session so configuration is applied once and per-step calls only carry
coordinates, cell data, atom numbers, and requested units.
~nwchemc_calculate_hessian~, ~nwchemc_calculate_dipole~,
~nwchemc_calculate_polarizability~, ~nwchemc_calculate_quadrupole~, and
~nwchemc_calculate_stress~ are one-shot wrappers around the same session
ForceInput validation and direct numeric geometry path.
The session owns the topology invariant. A caller that changes atom count or
species ordering is rejected before NWChem evaluation. Resetting or replacing
that topology is explicit: create a separate session, or add a dedicated reset
entry point with the same explicit semantics.
~nwchemc_session_set_params~ replaces the installed ~NWChemParams~ only before
the first accepted topology. After a session has accepted a step, parameter
replacement is rejected so stale direct RTDB keys cannot silently carry into a
new configuration.
Do not add a second key-value database for session or parameter state. Session
objects may own decoded scalar fields that are derived from the installed
~NWChemParams~ message. Any broader in-memory memo for parsed Cap'n Proto bytes
must use xxHash over the serialized bytes with the byte length and schema
identity in the key, and it must confirm byte equality before reusing a cached
decode.

The session path writes geometry and cell data numerically. Maintain that shape
for coordinates, atom numbers, and cell vectors. Text input is acceptable only
for NWChem grammar that does not have a confirmed direct API or RTDB key.
Text-valued ~NWChemSetDirective~ stanzas are already promoted to direct
~rtdb_cput~ calls in the embed path, so keep simple RTDB string options out of
~nw_inp_from_character~.
Typed/list ~NWChemSetDirective.valueType~ + ~values~ entries are also promoted:
~text~ uses ~rtdb_cput~, ~double~ uses ~MT_DBL~, ~integer~ uses ~MT_INT~, and
~logical~ uses ~MT_LOG~.

The result-carrier path keeps the native C force buffer ABI available for
low-overhead callers and gives RPC-style callers the same ~PotentialResult~
message used by rgpot. Stress result carriers write ~PotentialResult.stress~
in ~ForceInput.energyUnit / ForceInput.lengthUnit^3~; native stress buffers
keep NWChem atomic stress units.

Embed rendering omits explicit ~NWChemTaskStanza~ text. Full-deck rendering
keeps ~task theory operation~ lines, but C ABI evaluation writes
~task:theory~ and ~task:operation~ directly from the requested entry point
before calling the corresponding NWChem task routine.
The dipole entry points set ~task:operation~ to ~energy~ and extract the dipole
from RTDB instead of rendering a text property task.
The polarizability entry points set ~task:operation~ to ~property~ after the
energy task, install AO response defaults only for unset RTDB keys, and extract
~aoresponse:alpha~ from RTDB.
The quadrupole entry points set ~task:operation~ to ~energy~ and call
~hnd_mtpole2~ through loaded RTDB geometry and basis handles instead of
rendering a text property task.
The stress entry points set ~task:operation~ to ~stress~ and read the NWPW
stress RTDB arrays directly. The real PSPW stress smoke covers one-shot and
session raw buffers plus ~PotentialResult.stress~ serialization.

Simple named ~exchangeCorrelation~ values also reset ~pspw:SIC_all~,
~pspw:HFX~, ~band:HFX~, ~pspw:SIC_xc_parameter~, ~pspw:SIC_h_parameter~,
~pspw:HFX_parameter~, and ~band:HFX_parameter~ to the defaults used by the
NWPW input handler for named functionals. The nested
~exchange_correlation new ...~ grammar remains in the text/directive path.

Direct-path parity is covered by ~nwchem-config-parity~ when NWChem embed is
available. It compares ~NWChemParams.inputBlocks~ against
~PotentialConfig.nwchem~ structured DFT for the same H2 force input and catches
RTDB state drift between one-shot config installs.

** DONE Simulation Cell Direct Path
~NWChemSimulationCellStanza~ renders full-deck ~nwpw / simulation_cell~ text
and embed builds promote stable RTDB state directly through
~nwchemc_embed_set_rtdb_values~. The direct path writes ~<cell>:boundry~,
~<cell>:unita~, ~<cell>:ngrid~, ~<cell>:ngrid_small~, ~<cell>:box_delta~,
~<cell>:box_orient~, and ~<cell>:box_type~. The Cap'n Proto vector field is
~latticeVectorsBohr~ and must contain zero or nine values. Tests:
~test_parser_renders_structured_input~ and
~test_embed_config_uses_direct_dft_values~ cover full rendering, embed text
omission, and typed RTDB values.
Compact lattice directives (~sc~, ~fcc~, and ~bcc~) also promote directly to
~<cell>:unita~ from ~latticeKind~ plus ~latticeLengthBohr~, using the same
primitive-vector assignments as NWChem's ~simulation_cell_input.F~ parser.

** Expand Direct RTDB Settings
Move additional scalar options that have stable NWChem RTDB keys out of rendered
text stanzas. Good candidates are additional module scalar controls with
confirmed RTDB keys.

Each migrated option needs a cmocka parser test, a real NWChem test that proves
the direct path still computes finite outputs, and a reference to the upstream
NWChem input routine or RTDB key being matched.

** DONE Pseudopotential Entry Direct Path
~NWChemPseudopotentialStanza~ carries ~library~, ~pspw_library~,
~paw_library~, ~cpi~, ~teter~, ~allElements~ entries, and structured
~pspSpin~ / ~spinRules~. The C layer walks library entries with
~nwchemc_params_for_each_direct_pseudopotential~ and passes generated Cap'n
Proto entry values through the embed setup path. Fixed-width records exist only
at the final C/Fortran call into ~nwchemc_embed_set_pseudopotentials~. Embed
rendering omits the structured library entries and promotes
~pspSpin = disabled~ to the direct ~nwpw:pspspin~ and
~nwpw:pspspin_count~ RTDB reset keys. Indexed ~pspspin up/down~ spin-scaling
rules are walked with ~nwchemc_params_for_each_direct_pseudopotential_spin_rule~
as generated c-capnproto structs and promoted to typed RTDB values.
~semicoreSmall~ promotes to the ~nwpw:psp:semicore_small~ logical RTDB key
read by NWPW semicore force and stress kernels. Extra directives stay in text.
~uterm~ and ~utermRules~ promote to the direct indexed NWPW RTDB keys instead
of relying on rendered pseudopotential text.

The fixed-form bridge writes:

| Entry type | RTDB keys |
|------------+-----------|
| ~library~ / ~pspw_library~ | ~<element>:pspw_library~, ~<element>:pspw_library_type~ = 0, ~<element>:psp_reset~ = true |
| ~paw_library~ | ~<element>:paw_library~, ~<element>:psp_reset~ = true |
| ~cpi~ | ~<element>:pspw_library~, ~<element>:pspw_library_type~ = 2, ~<element>:psp_reset~ = true |
| ~teter~ | ~<element>:pspw_library~, ~<element>:pspw_library_type~ = 3, ~<element>:psp_reset~ = true |
| ~allElements~ | same keys with ~*~ as the element target |
| ~pspSpin = disabled~ | ~nwpw:pspspin~ = false, ~nwpw:pspspin_count~ = 0 |
| ~semicoreSmall~ | ~nwpw:psp:semicore_small~ logical |
| ~spinRules[]~ | ~nwpw:pspspin~ = true, ~nwpw:pspspin_count~, ~nwpw:pspspin_iamup:_000001~, channel-specific ~scale~, ~l~, optional ~m~, and ~ions~ keys |
| ~uterm = disabled~ | ~nwpw:uterm~ = false, ~nwpw:nuterms~ = 0 |
| ~utermRules[]~ | ~nwpw:uterm~ = true, ~nwpw:nuterms~, ~nwpw:uterm_scale:_000001~, ~nwpw:jterm_scale:_000001~, ~nwpw:uterm_l:_000001~, and ~nwpw:uterm_ions:_000001~ |

The ~nwchem-pseudopotential-rtdb~ integration test initializes a real NWChem
RTDB, calls ~nwchemc_store_pseudopotentials~ for library entries, applies the
typed-set pseudopotential payload through ~nwchemc_apply_direct_typed_sets~, and
reads back the exact ~pspw_library~, ~paw_library~, ~pspw_library_type~,
~psp_reset~, ~pspspin~, ~semicore_small~, and ~uterm~ keys. The test covers
~library~, ~pspw_library~, ~paw_library~, ~cpi~, ~teter~, ~allElements~,
~pspspin~, ~semicoreSmall~, and ~utermRules~ entries without routing through
NWChem text input.

The ~nwchem-pseudopotential-text-rtdb~ probe covers top-level text-input
parity for the same NWPW pseudopotential stanza against an NWPW-enabled NWChem
link. It renders the full ~nwpw / pseudopotentials~ block, parses it with
~nw_inp_from_character~, and reads the same library, reset, ~pspspin~,
~not_m~, semicore, and ~uterm~ RTDB keys that the direct path writes.
The ~nwchem-pspw-pseudopotential-forces~ smoke runs a Si PSPW force step whose
structured pseudopotential entry matches the Si atoms in ~ForceInput~ and
verifies finite ~PotentialResult.energy~ and ~PotentialResult.forces~ values
through one-shot and session result-carrier entry points.
The ~nwchem-nwpw-rtdb~ probe also installs a configured
~PotentialConfig.nwchem~ fixture and verifies the same NWPW control families
against the embedded RTDB handle, including string-valued ~vfieldFilenames~
and ~dosFilename~ controls, DOS scalars, director/FEI state, auxiliary
potentials, and NWPW multiplicity fan-out.

** Atom And Geometry Limits
Direct geometry and library-basis setup use ~n_atoms~-sized storage instead of
an embed-local atom cap. The C-facing Fortran allocates per-call arrays, and the
fixed-form RTDB helpers allocate geometry/basis tag storage from ~n_atoms~
before calling NWChem geometry/basis routines. The
~nwchem-geometry-basis-rtdb~ probe stores a 65-atom geometry and library basis
directly in RTDB without running SCF. Atomic-number to element-symbol mapping
uses the full periodic table through oganesson for direct geometry and basis
setup.

** ABI Surface
Keep adding calculation entry points around the same ~NWChemParams~ carrier:
properties beyond total dipole, optimized geometries, vibrational analysis, and
any rgpot-relevant potential/force forms. Avoid parallel option structs.

** Build Model
The embed build consumes an NWChem build tree because it needs source headers,
private module headers, static archives, ~stubs.o~, and GA config metadata. A
conda-forge runtime package is useful only if it exposes equivalent embed
artifacts and link metadata.

* C ABI
Include ~nwchemc.h~ and pass the serialized ~NWChemParams~ message bytes to the
ABI functions.

#+begin_src c
#include <nwchemc.h>
#include <stdlib.h>

int rc = nwchemc_set_params(params_bytes, params_size);
NWChemCResult result = nwchemc_energy_gradient(
    n_atoms, positions_ang, atomic_numbers, params_bytes, params_size,
    gradient_h_bohr);
NWChemCResult hessian = nwchemc_hessian(
    n_atoms, positions_ang, atomic_numbers, params_bytes, params_size,
    hessian_h_bohr2);
double dipole_au[3];
NWChemCResult dipole = nwchemc_dipole(
    n_atoms, positions_ang, atomic_numbers, params_bytes, params_size,
    dipole_au);
double quadrupole_au[6];
NWChemCResult quadrupole = nwchemc_quadrupole(
    n_atoms, positions_ang, atomic_numbers, params_bytes, params_size,
    quadrupole_au);
#+end_src

~params_bytes~ is an unpacked flat Cap'n Proto message.
It can come from pycapnp, a memory mapped file, rgpot, or another Cap'n Proto
binding that writes the standard flat stream format.

* Session Step Calls
Callers that drive multiple geometry steps can keep the NWChem configuration in
a session and pass a serialized ~ForceInput~ message for each step. The same
step carrier works for force, serialized result, Hessian, dipole, and
quadrupole entry points.

#+begin_src c
NWChemCSession *session = nwchemc_session_create(params_bytes, params_size);

if (session != NULL) {
  double forces_h_bohr[n_atoms * 3];
  size_t potential_result_capacity =
      nwchemc_potential_result_size_for_force_input(
          force_input_bytes, force_input_size);
  unsigned char *potential_result_bytes =
      malloc(potential_result_capacity);
  size_t potential_result_size = 0;
  double hessian_h_bohr2[(n_atoms * 3) * (n_atoms * 3)];
  NWChemCResult step = nwchemc_session_calculate_forces(
      session, force_input_bytes, force_input_size, forces_h_bohr,
      n_atoms * 3);
  NWChemCResult rpc_step = nwchemc_session_calculate_result(
      session, force_input_bytes, force_input_size,
      potential_result_bytes, potential_result_capacity,
      &potential_result_size);
  NWChemCResult curvature = nwchemc_session_calculate_hessian(
      session, force_input_bytes, force_input_size, hessian_h_bohr2,
      (n_atoms * 3) * (n_atoms * 3));
  double dipole_au[3];
  NWChemCResult property = nwchemc_session_calculate_dipole(
      session, force_input_bytes, force_input_size, dipole_au, 3);
  double quadrupole_au[6];
  NWChemCResult multipole = nwchemc_session_calculate_quadrupole(
      session, force_input_bytes, force_input_size, quadrupole_au, 6);
  free(potential_result_bytes);
  nwchemc_session_destroy(session);
}
#+end_src

~force_input_bytes~ is an unpacked flat ~ForceInput~ message. The message carries
the atom count, atomic numbers, flat coordinate array, optional row-major 3x3
cell, and coordinate length unit. Coordinates and cell vectors are converted to
the Angstrom values used by the C ABI boundary before the Fortran bridge stores
them in NWChem geometry state. The energy in ~NWChemCResult.energy_h~ is in
Hartree. ~forces_h_bohr~ is filled in Hartree per Bohr for low-overhead native
callers. ~nwchemc_session_calculate_result~ fills an unpacked flat
~PotentialResult~ Cap'n Proto message: ~energy~ uses ~ForceInput.energyUnit~,
and ~forces~ uses ~ForceInput.energyUnit / ForceInput.lengthUnit~.

The first successful session evaluation establishes the session topology: atom
count and the ordered atomic-number list. Later session calls may change
coordinates, units, and cell vectors, but they must keep the same topology.
Changing species or atom count is rejected by the active session and requires a
separate ~NWChemCSession~. ~nwchemc_session_set_params~ is a pre-topology
replacement hook; after the first successful evaluation, a new configuration
also requires a separate session.

~nwchemc_potential_result_size_for_force_input~ parses the step message and
returns the result byte count without evaluating NWChem. If the output buffer is
too small, ~nwchemc_session_calculate_result~ reports the required byte count
through ~potential_result_size~ and does not evaluate NWChem. ~hessian_h_bohr2~
is filled in Hartree per Bohr squared. ~dipole_au~ is filled with a
three-element total dipole vector in atomic units. ~quadrupole_au~ is filled
with six total traceless quadrupole tensor components in atomic units, ordered
~xx, xy, xz, yy, yz, zz~.

For a single geometry step, ~nwchemc_calculate_result~ accepts
~NWChemParams~, one ~ForceInput~, and the caller-owned ~PotentialResult~ output
buffer. It uses the same result-carrier path as
~nwchemc_session_calculate_result~ and returns the required output byte count
without initializing NWChem when the buffer is too small.
~nwchemc_calculate_hessian~ and ~nwchemc_calculate_dipole~ provide the same
one-shot ~NWChemParams~ + ~ForceInput~ carrier for native Hessian and dipole
buffers. ~nwchemc_calculate_quadrupole~ provides the same carrier for the
six-component quadrupole buffer.

* Parameter Ownership
~nwchemc~ reads the message with generated ~capnp-c~ bindings.
The public ABI does not require a C++ runtime and does not define a second
configuration structure.
Callers own the serialized bytes.
The library copies only the C strings needed by the Fortran bridge when it
applies the active NWChem configuration, plus numeric per-step geometry scratch
for session step calls.

* Result Units
The energy returned by ~nwchemc_energy_gradient()~ is in Hartree.
The gradient buffer is filled with nuclear gradients in Hartree per Bohr.
~nwchemc_energy_forces()~ and ~nwchemc_session_calculate_forces()~ return forces
as negative gradients in Hartree per Bohr.
~nwchemc_session_calculate_result()~ writes ~PotentialResult.energy~ and
~PotentialResult.forces~ in the units requested by ~ForceInput.energyUnit~ and
~ForceInput.lengthUnit~, while ~NWChemCResult.energy_h~ remains Hartree.
~nwchemc_hessian()~, ~nwchemc_session_calculate_hessian()~, and
~nwchemc_calculate_hessian()~ fill a dense row-major Cartesian Hessian in
Hartree per Bohr squared, with length ~(natoms * 3) * (natoms * 3)~.
~nwchemc_dipole()~, ~nwchemc_session_dipole()~, and
~nwchemc_session_calculate_dipole()~, and ~nwchemc_calculate_dipole()~ fill a
three-element total dipole vector in atomic units and keep
~NWChemCResult.energy_h~ in Hartree.
~nwchemc_quadrupole()~, ~nwchemc_session_quadrupole()~,
~nwchemc_session_calculate_quadrupole()~, and ~nwchemc_calculate_quadrupole()~
fill six total traceless quadrupole components in atomic units as
~xx, xy, xz, yy, yz, zz~ and keep ~NWChemCResult.energy_h~ in Hartree.
Input coordinates are Angstrom values in a flat ~[natoms * 3]~ array.

* Language Bindings
A binding only needs to produce the Cap'n Proto message and call the C ABI.
For Python, a typical path is:

1. Load ~schema/Potentials.capnp~ with pycapnp.
2. Fill an ~NWChemParams~ message.
3. Write the message as an unpacked flat byte stream.
4. Pass the byte pointer and length through ~ctypes~, ~cffi~, or a native
   extension module.

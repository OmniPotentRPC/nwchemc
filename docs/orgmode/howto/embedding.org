* C ABI
Include ~nwchemc.h~ and pass the serialized ~NWChemParams~ message bytes to the
ABI functions.

#+begin_src c
#include <nwchemc.h>
#include <stdlib.h>

int rc = nwchemc_set_params(params_bytes, params_size);
NWChemCResult result = nwchemc_energy_gradient(
    n_atoms, positions_ang, atomic_numbers, params_bytes, params_size,
    gradient_h_bohr);
NWChemCResult hessian = nwchemc_hessian(
    n_atoms, positions_ang, atomic_numbers, params_bytes, params_size,
    hessian_h_bohr2);
double dipole_au[3];
NWChemCResult dipole = nwchemc_dipole(
    n_atoms, positions_ang, atomic_numbers, params_bytes, params_size,
    dipole_au);
double quadrupole_au[6];
NWChemCResult quadrupole = nwchemc_quadrupole(
    n_atoms, positions_ang, atomic_numbers, params_bytes, params_size,
    quadrupole_au);
double stress_au[9];
NWChemCResult stress = nwchemc_stress(
    n_atoms, positions_ang, atomic_numbers, params_bytes, params_size,
    stress_au);
#+end_src

~params_bytes~ is an unpacked flat Cap'n Proto message.
It can come from pycapnp, a memory mapped file, rgpot, or another Cap'n Proto
binding that writes the standard flat stream format.

* Build and Verification
A real embed build needs an NWChem source/build tree, not only the ~nwchem~
runtime executable. ~-Dnwchem_root~ must point at the tree that provides the
configuration header, Global Arrays configuration helper, NWChem object files,
source include directories, and static archives:

- ~src/config/nwchem_config.h~
- ~src/tools/install/bin/ga-config~
- ~src/stubs.o~
- source include directories under ~src/~
- archives under ~lib/$NWCHEM_TARGET~

Build ~src/stubs.o~ through NWChem if the object is missing:

#+begin_src bash
export NWCHEM_TOP=/path/to/nwchem
export NWCHEM_TARGET=LINUX64
make -C "$NWCHEM_TOP/src" stubs.o
#+end_src

Many NWChem builds provide non-PIC static archives. Build ~nwchemc~ as a static
library when the selected archives are not suitable for a shared object:

#+begin_src bash
meson setup build-nwchem \
  -Dwith_nwchem=true \
  -Dnwchem_root="$NWCHEM_TOP" \
  -Dnwchem_target="$NWCHEM_TARGET" \
  -Dwith_tests=true \
  -Ddefault_library=static
meson compile -C build-nwchem
meson test -C build-nwchem --print-errorlogs
#+end_src

If the selected NWChem build requires an MPI launcher for in-process tests, set
the test rank count at configure time. The ~OMPI_MCA_*~ variables are Open MPI
transport controls and are only needed on hosts that require them:

#+begin_src bash
meson setup build-nwchem \
  -Dwith_nwchem=true \
  -Dnwchem_root="$NWCHEM_TOP" \
  -Dnwchem_target="$NWCHEM_TARGET" \
  -Dwith_tests=true \
  -Ddefault_library=static \
  -Dnwchem_test_mpi_ranks=2
env OMPI_MCA_pml=ob1 OMPI_MCA_btl=self,tcp \
  meson test -C build-nwchem --print-errorlogs --num-processes 2
#+end_src

The build reads ~NW_MODULE_LIBS~ from ~nwchem_config.h~, Global Arrays linker
flags from ~ga-config~, and optional support archives from the NWChem library
directory. ~libperfm~ is always requested; ~libpeigs~ and ~libpeigs_comm~ are
added only when the corresponding archive exists.

Use the comparison driver to check the C ABI results against the NWChem CLI
for the integration cases under ~tests/integration/nw/~:

#+begin_src bash
python3 tools/compare_nwchem_cli.py \
  --out-dir build/nwchem-compare-embed \
  --embed-build build-nwchem \
  --require-cli \
  --require-embed
#+end_src

When both the CLI and embed test leg need an MPI launcher, pass both launcher
commands explicitly:

#+begin_src bash
python3 tools/compare_nwchem_cli.py \
  --out-dir build/nwchem-compare-embed \
  --embed-build build-nwchem \
  --require-cli \
  --require-embed \
  --nwchem-command "mpirun -np 2 $NWCHEM_TOP/bin/$NWCHEM_TARGET/nwchem" \
  --embed-command "mpirun -np 2"
#+end_src

The comparison report is written as ~nwchem_compare_report.txt~ and
~nwchem_compare_report.json~ in the selected output directory. The embed leg
writes ~NWCHEMC_COMPARE_JSON~ through the real C ABI test binary so the report
captures the embedded energy and gradient vector.

* Session Step Calls
Callers that drive multiple geometry steps can keep the NWChem configuration in
a session and pass a serialized ~ForceInput~ message for each step. The same
step carrier works for force, serialized result, Hessian, dipole, and
quadrupole, and stress entry points.

#+begin_src c
NWChemCSession *session = nwchemc_session_create(params_bytes, params_size);

if (session != NULL) {
  double forces_h_bohr[n_atoms * 3];
  size_t potential_result_capacity =
      nwchemc_potential_result_size_for_force_input(
          force_input_bytes, force_input_size);
  unsigned char *potential_result_bytes =
      malloc(potential_result_capacity);
  size_t potential_result_size = 0;
  double hessian_h_bohr2[(n_atoms * 3) * (n_atoms * 3)];
  NWChemCResult step = nwchemc_session_calculate_forces(
      session, force_input_bytes, force_input_size, forces_h_bohr,
      n_atoms * 3);
  NWChemCResult rpc_step = nwchemc_session_calculate_result(
      session, force_input_bytes, force_input_size,
      potential_result_bytes, potential_result_capacity,
      &potential_result_size);
  NWChemCResult curvature = nwchemc_session_calculate_hessian(
      session, force_input_bytes, force_input_size, hessian_h_bohr2,
      (n_atoms * 3) * (n_atoms * 3));
  double dipole_au[3];
  NWChemCResult property = nwchemc_session_calculate_dipole(
      session, force_input_bytes, force_input_size, dipole_au, 3);
  double quadrupole_au[6];
  NWChemCResult multipole = nwchemc_session_calculate_quadrupole(
      session, force_input_bytes, force_input_size, quadrupole_au, 6);
  double stress_au[9];
  NWChemCResult cell_stress = nwchemc_session_calculate_stress(
      session, force_input_bytes, force_input_size, stress_au, 9);
  free(potential_result_bytes);
  nwchemc_session_destroy(session);
}
#+end_src

~force_input_bytes~ is an unpacked flat ~ForceInput~ message. The message carries
the atom count, atomic numbers, flat coordinate array, optional row-major 3x3
cell, and coordinate length unit. Coordinates and cell vectors are converted to
the Angstrom values used by the C ABI boundary before the Fortran bridge stores
them in NWChem geometry state. ~ForceInput.hasCharge~ / ~ForceInput.charge~ and
~ForceInput.hasMultiplicity~ / ~ForceInput.multiplicity~ override the session
charge or spin multiplicity for that serialized step; unset flags keep the
values from ~NWChemParams~, and non-positive multiplicities are rejected. The
energy in ~NWChemCResult.energy_h~ is in Hartree. ~forces_h_bohr~ is filled in
Hartree per Bohr for low-overhead native callers.
~nwchemc_session_calculate_result~ fills an unpacked flat
~PotentialResult~ Cap'n Proto message: ~energy~ uses ~ForceInput.energyUnit~,
and ~forces~ uses ~ForceInput.energyUnit / ForceInput.lengthUnit~.
~nwchemc_session_calculate_hessian_result~ fills ~PotentialResult.hessian~ with
the dense row-major Cartesian Hessian in
~ForceInput.energyUnit / ForceInput.lengthUnit^2~.
~nwchemc_session_calculate_dipole_result~ and
~nwchemc_session_calculate_quadrupole_result~ fill ~PotentialResult.dipole~ and
~PotentialResult.quadrupole~ in atomic units.
~nwchemc_session_calculate_stress_result~ fills ~PotentialResult.stress~ with
the row-major 3x3 stress tensor in
~ForceInput.energyUnit / ForceInput.lengthUnit^3~.
~nwchemc_session_calculate_optimize_result~ fills
~PotentialResult.optimizedPos~ in ~ForceInput.lengthUnit~ and
~PotentialResult.energy~ in ~ForceInput.energyUnit~.
~nwchemc_session_calculate_frequencies_result~ fills
~PotentialResult.frequencies~ in cm^-1 and ~PotentialResult.intensities~ in
atomic units.

The first successful session evaluation establishes the session topology: atom
count and the ordered atomic-number list. Later session calls may change
coordinates, units, and cell vectors, but they must keep the same topology.
Changing species or atom count is rejected by the active session and requires a
separate ~NWChemCSession~. ~nwchemc_session_set_params~ is a pre-topology
replacement hook; after the first successful evaluation, a new configuration
also requires a separate session.

~nwchemc_potential_result_size_for_force_input~ parses the step message and
returns the result byte count without evaluating NWChem. If the output buffer is
too small, ~nwchemc_session_calculate_result~ reports the required byte count
through ~potential_result_size~ and does not evaluate NWChem.
~nwchemc_hessian_result_size_for_force_input~ provides the same preflight size
for Hessian result messages. ~nwchemc_dipole_result_size_for_force_input~ and
~nwchemc_quadrupole_result_size_for_force_input~ provide the fixed-size
property result preflight. ~nwchemc_stress_result_size_for_force_input~
provides the fixed-size stress result preflight.
~nwchemc_optimize_result_size_for_force_input~ and
~nwchemc_frequencies_result_size_for_force_input~ provide the same preflight
contract for optimized coordinates and frequency/intensity result messages.
~hessian_h_bohr2~ is filled in Hartree per Bohr squared. ~dipole_au~ is filled
with a three-element total dipole vector in atomic units. ~quadrupole_au~ is
filled with six total traceless quadrupole tensor components in atomic units,
ordered ~xx, xy, xz, yy, yz, zz~.
~stress_au~ is filled with nine row-major 3x3 stress tensor components in
NWChem atomic stress units.

For a single geometry step, ~nwchemc_calculate_result~ accepts
~NWChemParams~, one ~ForceInput~, and the caller-owned ~PotentialResult~ output
buffer. It uses the same result-carrier path as
~nwchemc_session_calculate_result~ and returns the required output byte count
without initializing NWChem when the buffer is too small.
~nwchemc_session_calculate_energy~ and ~nwchemc_calculate_energy~ return only
the native Hartree scalar for serialized ~ForceInput~ callers that do not need
forces or a ~PotentialResult~ message.
~nwchemc_calculate_hessian~ and ~nwchemc_calculate_dipole~ provide the same
one-shot ~NWChemParams~ + ~ForceInput~ carrier for native Hessian and dipole
buffers. ~nwchemc_calculate_forces~ provides the same carrier for native force
buffers. ~nwchemc_calculate_quadrupole~ provides the same carrier for the
six-component quadrupole buffer. ~nwchemc_calculate_stress~ provides the same
carrier for the nine-component stress buffer.
~nwchemc_calculate_hessian_result~ uses the same one-shot carrier and writes
the Hessian to ~PotentialResult.hessian~.
~nwchemc_calculate_dipole_result~ and
~nwchemc_calculate_quadrupole_result~ write the corresponding property lists to
~PotentialResult~. ~nwchemc_calculate_stress_result~ writes
~PotentialResult.stress~. ~nwchemc_calculate_optimize_result~ and
~nwchemc_calculate_frequencies_result~ write optimized coordinates and
frequency/intensity lists through the same one-shot result-carrier contract.

* Parameter Ownership
~nwchemc~ reads the message with generated ~capnp-c~ bindings.
The public ABI does not require a C++ runtime and does not define a second
configuration structure.
Callers own the serialized bytes.
The library copies only the C strings needed by the Fortran bridge when it
applies the active NWChem configuration, plus numeric per-step geometry scratch
for session step calls.

* Result Units
The energy returned by ~nwchemc_energy_gradient()~ is in Hartree.
The gradient buffer is filled with nuclear gradients in Hartree per Bohr.
~nwchemc_session_calculate_energy()~ and ~nwchemc_calculate_energy()~ return
their scalar energy in Hartree.
~nwchemc_energy_forces()~, ~nwchemc_session_calculate_forces()~, and
~nwchemc_calculate_forces()~ return forces as negative gradients in Hartree per
Bohr.
~nwchemc_session_calculate_result()~ writes ~PotentialResult.energy~ and
~PotentialResult.forces~ in the units requested by ~ForceInput.energyUnit~ and
~ForceInput.lengthUnit~, while ~NWChemCResult.energy_h~ remains Hartree.
~nwchemc_hessian()~, ~nwchemc_session_calculate_hessian()~, and
~nwchemc_calculate_hessian()~ fill a dense row-major Cartesian Hessian in
Hartree per Bohr squared, with length ~(natoms * 3) * (natoms * 3)~.
~nwchemc_session_calculate_hessian_result()~ and
~nwchemc_calculate_hessian_result()~ write that same Hessian to
~PotentialResult.hessian~ after conversion to
~ForceInput.energyUnit / ForceInput.lengthUnit^2~.
~nwchemc_dipole()~, ~nwchemc_session_dipole()~,
~nwchemc_session_calculate_dipole()~, and ~nwchemc_calculate_dipole()~ fill a
three-element total dipole vector in atomic units and keep
~NWChemCResult.energy_h~ in Hartree.
~nwchemc_session_calculate_dipole_result()~ and
~nwchemc_calculate_dipole_result()~ write the same vector to
~PotentialResult.dipole~.
~nwchemc_quadrupole()~, ~nwchemc_session_quadrupole()~,
~nwchemc_session_calculate_quadrupole()~, and ~nwchemc_calculate_quadrupole()~
fill six total traceless quadrupole components in atomic units as
~xx, xy, xz, yy, yz, zz~.
~nwchemc_session_calculate_quadrupole_result()~ and
~nwchemc_calculate_quadrupole_result()~ write those components to
~PotentialResult.quadrupole~.
~nwchemc_stress()~, ~nwchemc_session_stress()~,
~nwchemc_session_calculate_stress()~, and ~nwchemc_calculate_stress()~ fill a
nine-component row-major 3x3 stress tensor in NWChem atomic stress units.
~nwchemc_session_calculate_stress_result()~ and
~nwchemc_calculate_stress_result()~ write that tensor to
~PotentialResult.stress~ after conversion to
~ForceInput.energyUnit / ForceInput.lengthUnit^3~.
~nwchemc_optimize()~, ~nwchemc_session_optimize()~,
~nwchemc_session_calculate_optimize()~, and ~nwchemc_calculate_optimize()~ fill
optimized Cartesian coordinates in Angstrom and keep ~NWChemCResult.energy_h~
in Hartree.
~nwchemc_session_calculate_optimize_result()~ and
~nwchemc_calculate_optimize_result()~ write optimized coordinates to
~PotentialResult.optimizedPos~ in ~ForceInput.lengthUnit~ and write
~PotentialResult.energy~ in ~ForceInput.energyUnit~.
~nwchemc_frequencies()~, ~nwchemc_session_frequencies()~,
~nwchemc_session_calculate_frequencies()~, and
~nwchemc_calculate_frequencies()~ fill harmonic frequencies in cm^-1 and
optional IR intensities in atomic units.
~nwchemc_session_calculate_frequencies_result()~ and
~nwchemc_calculate_frequencies_result()~ write those arrays to
~PotentialResult.frequencies~ and ~PotentialResult.intensities~.
Input coordinates are Angstrom values in a flat ~[natoms * 3]~ array.

* Language Bindings
A binding only needs to produce the Cap'n Proto message and call the C ABI.
For Python, a typical path is:

1. Load ~schema/Potentials.capnp~ with pycapnp.
2. Fill an ~NWChemParams~ message.
3. Write the message as an unpacked flat byte stream.
4. Pass the byte pointer and length through ~ctypes~, ~cffi~, or a native
   extension module.
